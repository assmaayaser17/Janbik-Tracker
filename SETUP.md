# جنبك — لوحة تتبّع البايلوت (Next.js + Supabase)

تطبيق Next.js جاهز، متصل بقاعدة بيانات مباشرة (Supabase). أي تعديل من أي عضو فريق يظهر فورًا لكل من يفتح اللينك. تم تجربة البناء (`next build`) بنجاح.

## 1) Supabase (قاعدة البيانات + تسجيل الدخول)

1. سجّلي مجانًا في [supabase.com](https://supabase.com) وأنشئي مشروع جديد.
2. من القائمة الجانبية: **SQL Editor → New query**، الصقي محتوى `supabase/schema.sql` بالكامل واضغطي **Run**. هذا ينشئ الجدول، صلاحيات الأمان (القراءة للجميع، الكتابة للمسجّلين فقط)، والـ 54 حضانة الحالية كبيانات أولية.
3. من **Authentication → Users → Add user**، أضيفي حساب لكل عضو فريق (إيميل + كلمة مرور). هذول هم اللي هيقدروا يعدّلوا اللوحة. مفيش تسجيل ذاتي مفتوح للعامة — الحسابات تتضاف يدويًا فقط.
4. من **Settings → API** انسخي: **Project URL** و **anon public key**.

## 2) التجربة محليًا (اختياري)

```bash
cp .env.local.example .env.local
# افتحي .env.local وحطي فيها الـ URL والـ anon key من الخطوة السابقة
npm install
npm run dev
```
افتحي http://localhost:3000

## 3) النشر أونلاين — Vercel (مجاني)

Next.js من إنتاج نفس شركة Vercel، فهو أسهل استضافة مجانية له (بديل GitHub Pages/Netlify اللي مش بيدعموا Next.js بنفس السهولة).

1. ارفعي مجلد المشروع كـ repo على GitHub (خاص أو عام).
2. سجّلي دخول على [vercel.com](https://vercel.com) بحساب GitHub → **Add New → Project** → اختاري الـ repo.
3. في خطوة الإعداد، أضيفي **Environment Variables**:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   (نفس القيم من خطوة Supabase)
4. اضغطي **Deploy**. هتاخد دقيقة وهيطلعلك لينك زي `janbik-tracker.vercel.app` جاهز تتشاركوه.
5. أي تعديل في الكود بعد كده وترفعيه على GitHub، Vercel هينشره تلقائي. أما تعديل البيانات نفسها (الحالات، الملاحظات...) فبيحصل مباشرة من الموقع نفسه، مفيش داعي لإعادة نشر.

## كيف الصلاحيات شغالة

- **أي حد عنده اللينك**: يفتح ويشوف اللوحة (وضع القراءة) بدون تسجيل دخول.
- **أعضاء الفريق**: يضغطوا "وضع التحرير" → يسجلوا دخول بالإيميل وكلمة المرور اللي انضافتلهم في Supabase → يقدروا يعدّلوا/يضيفوا/يحذفوا، والكل يشوف التحديث لحظيًا.
- تصدير Excel شغال دايمًا. استيراد Excel و"استعادة النسخة الموحّدة" بيستبدلوا القائمة بالكامل لكل المشاهدين — استخدميهم بحذر.

## هيكل المشروع

```
app/page.js          الصفحة الرئيسية (كل منطق اللوحة)
app/layout.js         إعداد الصفحة العام
app/globals.css       نفس تصميم النسخة الأصلية
lib/supabaseClient.js  الاتصال بـ Supabase
lib/pilotData.js       القوائم الثابتة + نسخة احتياطية من الـ 54 حضانة
supabase/schema.sql    السكربت اللي تشغّليه مرة واحدة في Supabase
```
